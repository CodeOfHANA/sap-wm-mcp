# Skill: /wm-abapgit-sync

Export all ABAP objects from the SAP system into `abap/src/` so they are tracked in Git and installable via abapGit.

## When to use
- After creating or modifying any ABAP object via VSP (`/wm-rap-scaffold`, `/wm-rap-cds`, `/wm-rap-behavior`)
- Before committing and pushing to GitHub
- When a customer reports a missing object (verify what's in the repo)

---

## What this does

Reads every ABAP object in package `ZWM_MCP` from the live SAP system using VSP `GetSource`, then writes each one to the correct file in `abap/src/` using abapGit naming conventions.

## Step-by-step

### 1. Read each object source via VSP

For CDS views and BDEF, use `GetSource`:
```
VSP GetSource: object type = DDLS, object name = ZR_WMSTORAGBIN
```

For ABAP classes, read both the global part and local types separately:
```
VSP GetSource: object type = CLAS, object name = ZBP_R_WMSTORAGBIN
VSP GetSource: object type = CLAS/local types, object name = ZBP_R_WMSTORAGBIN
```

For service binding and definition, these are XML-based ŌĆö use VSP `GetSource` with type SRVB/SRVD.

### 2. Write to `abap/src/`

Write the source content verbatim to the file. Do not modify the content ŌĆö abapGit expects the exact format that SAP generates.

### 3. Verify the package definition file exists

`abap/src/package.devc.xml` ŌĆö create if missing:

```xml
<?xml version="1.0" encoding="utf-8"?>
<asx:abap xmlns:asx="http://www.sap.com/abapxml" version="1.0">
 <asx:values>
  <DATA>
   <DEVCLASS>ZWM_MCP</DEVCLASS>
   <CTEXT>SAP Classic WM MCP Server ŌĆö RAP Service</CTEXT>
   <AS4USER>NOMANH</AS4USER>
   <PDEVCLASS>$TMP</PDEVCLASS>
  </DATA>
 </asx:values>
</asx:abap>
```

### 4. Commit

```bash
git add abap/src/
git commit -m "abapgit: export ZWM_MCP package objects after [what changed]"
git push
```

## abapGit file format reference

### CDS View (`.ddls.asddls`)
Plain ABAP CDS source ŌĆö exactly as shown in ADT. Example:
```
@AccessControl.authorizationCheck: #NOT_REQUIRED
@EndUserText.label: 'WM Storage Bin'
define view entity ZR_WMSTORAGBIN
  as select from lgpla
{ ... }
```

### BDEF (`.bdef.asbdef`)
Plain BDL source ŌĆö exactly as shown in ADT.

### ABAP Class global (`.clas.abap`)
```abap
CLASS zbp_r_wmstoragbin DEFINITION PUBLIC ABSTRACT FINAL
  FOR BEHAVIOR OF zr_wmstoragbin.
ENDCLASS.
CLASS zbp_r_wmstoragbin IMPLEMENTATION.
ENDCLASS.
```

### ABAP Class local types (`.clas.locals_imp.abap`)
The full local class section containing `lhc_` handler classes and their implementations.

### Service Binding (`.srvb.srvb`)
XML format generated by SAP ŌĆö read via VSP and write verbatim.

### DDIC Structure (`.stru.xml`)
XML format ŌĆö read via VSP `GetSource` with type STRU.

## How a customer installs from abapGit

```
1. SAP GUI ŌåÆ run report ZABAPGIT (or transaction /ZABAPGIT if registered)
2. Click "New Online" button
3. URL:     https://github.com/CodeOfHANA/sap-wm-mcp
4. Package: ZWM_MCP  ŌåÉ type this, abapGit will create it
5. Click "Pull"
6. Confirm transport request (assign to a transport or use $TMP for testing)
7. abapGit creates all objects from abap/src/ files
8. Publish service binding in /IWFND/V4_ADMIN or ADT
```

After Pull, the customer has the full RAP service. They then configure `.env` and run `npx sap-wm-mcp`.

## abapGit not yet installed on customer system

Point them to: https://github.com/abapGit/abapGit

Quickest install:
1. Download `abapgit.abap` from the releases page (single-file version)
2. In SAP GUI: SE38 ŌåÆ create report `ZABAPGIT`, paste the content, activate, run
3. That's it ŌĆö abapGit is now available as a report

No Basis involvement needed. Works on any ABAP 7.40+ system.
